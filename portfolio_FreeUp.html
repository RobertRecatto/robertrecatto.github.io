FreeUp



For this project, myself and a group of 3 others united to create an entire mobile web application from complete scratch in just 8 weeks.

This project gave us all a hastened yet thorough glance into the many intriciacies of the design process.

Our finished product delivers users with a convenient way to easily visualize when both they and their friends are free to meet up.

Project Features


Our application allows users to create an account and login using firebase and quickly enter their entire work schedule for a nice colorful view of it.

Users can also enter in friend's account names as well to superimpose their friends' schedules on top of their own.

The app allows for up to 5 schedules to be superimposed on top of eachother, and has convenient filters at the top to show and unshow certain schedules.

The scope of this project goes much deeper than just the implementation though as the first 4 weeks of the project consisted entirely of user testing and needfinding.

My Contributions


Many of my largest contributions for this particular project came before any code was even written.

I went out and interviewed people in needfinding efforts to ultimately help come up with the idea for our application.

I also played a large role in testing after programming had began as I hooked up our application to google analytics to conduct an A/B test to help choose between two different views. Ultimately my A/B test proved that a radio button implementation of the days of the week was superior to a drop down menu.

Most of my programmatic contributions to this project came on the JavaScript side as I used vanilla JS to write and algorithm to convert users entered times into a JSON object to easily show on the schedule page.

What I learned


This process was one of the most thorough hands on projects I've ever worked on so needless to say I was able to learn a lot from it.

From bootstrap and css for the UI to even some small backend work hooking it up to Firebase, this project had a lot to offer for a small team of 4.

The planning before the project was also extremely useful as we used almost a waterfall appraoch in preparing our app.

From storyboards to hand drawn prototypes, this project taught me how to find success when starting from scratch on a large project.
